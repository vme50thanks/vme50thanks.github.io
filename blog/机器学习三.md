### 循环神经网络RNN

B站视频里没有RNN的内容，但是配套书籍里有，还是稍微学习了一下

卷积神经网络CNN是用来解决图片问题的，而循环神经网络RNN是用来解决序列数据的，什么是序列数据呢？就是那些前后有依赖关系的数据。比如“我爱游戏”，如果没有RNN，机器会认为我爱游戏和游戏爱我没有任何区别，因为机器不知道我和游戏之间的序列关系。

#### 独热编码

机器不认识文字，所以要把单词用向量的方式表示出来。假设词典中有 5 个单词：apple，bag，cat，dog，elephant，向量的大小就是词典大小。每一个维度对应词典的一个单词。对应单词的维度为 1，其他为 0。

`apple = [1, 0, 0, 0, 0]`

`bag = [0, 1, 0, 0, 0]`

`cat = [0, 0, 1, 0, 0]`

`dog = [0, 0, 0, 1, 0]`

`elephant = [0, 0, 0, 0, 1]`

#### 什么是RNN？

普通神经网络只有输入和输出两种状态，而RNN会有输入—>隐状态—>输出三种状态，什么是隐状态呢？比如我—>爱—>游戏，当机器读到我时，就会把我记作h1，读到爱时，就会记作h2，h2里包含了h1和爱的信息，以此类推，无线套娃。

#### LSTM(RNN最重要的部分)

LSTM，全称Long Short-Term Memory——长短期记忆网络，顾名思义就是普通RNN记性差，LSTM新增记忆单元，让h层进行运算，c层负责长期记忆。LSTM分为三个门：遗忘门（决定忘记多少旧内容），输入门（决定用多少新内容），输出门（决定输出什么到隐藏状态）。

#### LSTM是如何解决梯度消失的问题

>
> 换成 LSTM？。
>
> A：LSTM 可以处理梯度消失的问题。用这边的式子回答看看。RNN 跟 LSTM 在面对
>
> 记忆元的时候，它处理的操作其实是不一样的。在 RNN 里面，在每一个时间点，神经
>
> 元的输出都要记忆元里面去，记忆元里面的值都是会被覆盖掉。但是在 LSTM 里面不
>
> 一样，它是把原来记忆元里面的值乘上一个值再把输入的值加起来放到单元里面。所
>
> 以它的记忆和输入是相加的。LSTM 和 RNN 不同的是，如果权重可以影响到记忆元
>
> 里面的值，一旦发生影响会永远都存在。而 RNN 在每个时间点的值都会被格式化掉，
>
> 所以只要这个影响被格式化掉它就消失了。但是在 LSTM 里面，一旦对记忆元造成影
>
> 响，影响一直会被留着，除非遗忘门要把记忆元的值洗掉。不然记忆元一旦有改变，只
>
> 会把新的东西加进来，不会把原来的值洗掉，所以它不会有梯度消失的问题。

### 自注意力机制

当输入不是一个固定向量，而是一串长度不固定的向量时，模型怎么处理整段序列的信息，自注意力机制就是讲这个。

然而为什么需要这个自注意力机制呢？事实上，很多任务的输入都是长度可变的向量序列，比如文字处理，可以看成一串词向量，一个图像可以看成很多像素的向量，比如句子：

> I saw a saw.

普通的模型根本分辨不出第一个saw是看，第二个saw是锯子，模型必须结合上下文来看，通过自注意力，让每个位置都能关注整个序列

简化成一句话：**每个位置都问：整个序列里谁对我最重要？然后按照重要程度把它们的信息加权汇总起来**

#### 输入是向量序列的情况

当输入是向量序列时，视频讲了三种输出情况

##### 输入和输出数量相同

举例：

- 词性标注：每个词都要判断词性；
- 语音识别中的帧分类：每一帧声音都要判断属于哪个音标；
- 图节点分类：每个节点都要输出一个类别。

这些举例都属于**序列标注**

##### 输入是一个序列，输出是一个标签

举例：

- 情感分析：输入一句话，输出“正面”或“负面”；
- 说话人识别：输入一段声音，判断是谁说的；
- 文本分类：输入一篇文章，判断类别。

##### 序列到序列

举例：

- 机器翻译；
- 语音识别；
- 聊天机器人；
- 语音合成。

#### 自注意力的运作原理

**以第一种输入输出为例，假设输入序列是：**

```
a1, a2, a3, a4
```

**自注意力会输出：**

```
b1, b2, b3, b4
```

**要注意：**

> **每个输出 bi 都不是只由 ai 得到的，而是综合考虑所有输入 a1 到 a4 后得到的。**

关于核心计算流程就不说了，挺恶心的，感觉也用不到，但是有3个最重要的向量要注意下：

``Query，Q，查询,表示我想找什么信息``

**``Key，K，键，我有什么信息可以被别人匹配``**

**``Value，V，值，真正要被提取和汇总的信息``**

反正自注意力的本质就是每个位置先问：“谁和我最相关？”然后按相关程度，把其他位置的信息吸收进来。

#### 多头注意力

多头自注意力的使用是非常广泛的，有一些任务，比如翻译、语音识别，用比较多的头可以得到比较好的结果。再比如在一句话里，一个词可能需要关注语法关系、 语义关系、 指代关系等关系，单头注意力只有一种关注方式，可能不够。

多头注意力其实就是使用多组 Q、K、V，每个头都会得到一个输出，最后把这些输出拼接起来，再经过一个矩阵变换，得到最终结果。

#### 位置编码

跟RNN一样，自注意力也要懂位置顺序，所以要给每个位置加上一个位置向量，即位置编码。

有几种设计方式：

- 可以用固定的正弦、余弦函数
- 也可以让模型自己学习位置向量
- 也可以设计更复杂的相对位置编码

#### 截断自注意力

截断自注意力可以处理向量序列长度过大的问题。截断自注意力在做自注意力的时候不要看一整句话，就只看一个小的范围就好，比如当前语音帧只看前后若干帧，这个范围是人设定的。在做自注意力的时候，也许没有必要让自注意力考虑一整个句子，只需要考虑一个小范围就好，这样就可以加快运算的速度。这就是截断自注意力。

#### 自注意力与CNN和RNN的对比

##### 与CNN对比

CNN 主要依赖局部感受野。  每个卷积核只看图像中的一个局部区域。这适合图像，因为图像中很多局部模式很重要，比如边缘、纹理、局部形状。自注意力可以让每个位置直接看整个输入。例如图像中一个像素或 patch，可以直接关注图像中任何其他位置。可以把 CNN 看成自注意力的一种受限版本。CNN 人为规定每个位置只能看局部范围，
自注意力则让模型自己学习哪些位置重要。
所以：CNN 限制更多，比较适合数据少的情况；自注意力更灵活，但通常需要更多数据。

##### 与RNN对比

RNN 处理序列时，是一步一步处理的，每一步依赖前一步的隐藏状态。这相当于模型有一个记忆，把前面的信息传到后面。然而如果最后一个位置要使用最前面的信息，信息必须一步步传过去，可能在中途逐渐丢失，并且后一个时间点依赖前一个时间点，所以必须按顺序计算，不能并行运算

而自注意力中，每个位置都可以直接和任意其他位置建立联系。比如第 100 个词可以直接关注第 1 个词。
